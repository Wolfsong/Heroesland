Квестовые тексты - почти обычный HTML-файл, загружаемый с сервера в клиент. К сожалению, встроенный парсер HTML у флеша убог и мы будет искать/делать другой. Пока используем то, что есть.
I. Допустимые теги:
<p></p> - параграф и выравнивание текста
<br>
<b></b>
<u></u>
<i></i>
<font></font> - цвет и размер, в том числе - относительный
<img> - картинка по ссылке. только с новой строки и только по левому или правому краям...
<a href></a> - ссылка как для открытия внешних ссылок, так и для выполнения внутренних команд

II. Предварительный парсинг текста перед показом:
1. Заменяем @@heroname@@ на имя героя. Потом будут уровень и всё, что хранится на клиенте., как потребуются.
2. Заменяем всё, что заключено между двумя двойными подчеркиваниями на вызов языковой функции. Например, __item::name::31__ замениться га слово "Золото".

III. Действия по кликам на ссылки:
1. <a href='event:unit?id=85'> - Показать при клике описание юнита с id=85 (Скелет)
2. <a href='event:item?id=45'> - Показать при клике описание предмета с id=45 (Топор Кентавра)
3. <a href='event:object?id=1140'> - Показать при клике описание объекта на карте с id типа объекта=1140 (Хижина Мага)
4. <a href='event:minimap?m=9^x=136^y=178'> - Пометить при клике на миникарте крестиком и стрелкой цель с координатами 136:178. карта #9
5. <a href='event:list'> - Показать при клике список квестов для текущего квестера
6. <a href='event:action?q=3^s=1^a=1'> - Отправить при клике на сервер команду: выполнить действие #1 для стадии #1 квеста #3

IV. Именование html-файлов:
<фракция>_quest_<номер квеста>_<действие>.html Например, good_quest_001_accept.html
Фракции:
good, evil, nuetral

V. Схема оформления квеста:
Создаем папку с названием квеста на английском. В папке должно быть 9 файлов обязательных, перечисленных ниже и названных по схеме IV.

1. start - показываем начальную страницу. предлагаем выбрать дейсвие
2. accept - показываем при принятии квеста героем
3. reject - показываем при отказе от принятия квеста героем
4. check - показываем при повторном заходе, когда квест принят. даем сказать, что герой готов или отказаться от квеста
5. cancel - показываем команду для отказа от квеста в самом верху описания квеста в окне героя
6. noncondition - показываем сообщение о том, что у героя еще не всё готово - если он пытается закончить квест
7. progress - показываем "запись в журнале" для незаконченного квеста. сверху будет содержимое файла "cancel", посередине - этот текст, внизу - количество выполненных заданий по квесту
8. success - показываем сообщение в случае успешного выполнения квеста
9. finish - показываем "запись в журнале" для выполненного квеста
